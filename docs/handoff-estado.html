3. Como validar no seu Windows

Rode na raiz do projeto. Estes comandos NÃO puderam ser executados no ambiente da sessão porque o node_modules foi instalado no Windows (binários nativos de rollup/esbuild são de outra plataforma) — mas rodam normal na sua máquina.

# 1. Lint (esperado: 0 errors)
npm run lint

# 2. Testes unitários (esperado: tudo verde, incl. tests/projecao.test.js)
npm test

# 3. Guardrail anti-XSS (esperado: ✓ 0 sinks sem escape)
npm run security:xss

# 4. Build de produção (gera dist/ com bundle único + CSP limpa)
npm run build
#    Confira depois:
#    - dist/index.html tem 1 <script defer src="js/app.bundle.js"> + pin-guard bloqueante
#    - dist/index.html NÃO contém "localhost" na CSP

# 5. Abra o app e teste o essencial na mão:
#    - login/PIN, criar transação, extrato, orçamento, dashboard
#    - o indicador de sync aparece no canto
Antes de commitar: o repositório já tinha trabalho seu não commitado. Rode git diff / git status para revisar as mudanças (minhas + as suas WIP) e commite em partes lógicas. Todos os arquivos que toquei passaram por node --check e o pacote inteiro por lint + 231 testes.
